================================================================================
KEY TECHNICAL DECISIONS & RATIONALE
================================================================================

1. Next.js 16 App Router
   Rationale: Server components for better performance, built-in API routes,
              streaming SSR, and improved developer experience

2. Supabase for Backend
   Rationale: PostgreSQL with real-time capabilities, built-in auth,
              row-level security, and generous free tier

3. TypeScript Strict Mode
   Rationale: Type safety prevents runtime errors, better IDE support,
              self-documenting code

4. Server Actions over API Routes
   Rationale: Simplified data mutations, automatic revalidation,
              better type inference, reduced boilerplate

5. Zod for Validation
   Rationale: Runtime type checking, schema-based validation,
              TypeScript integration, excellent error messages

6. Tailwind CSS
   Rationale: Utility-first approach, consistent design system,
              small bundle size, rapid prototyping

CHALLENGES ENCOUNTERED & SOLUTIONS
================================================================================

Challenge 1: Middleware Not Checking Admin Role
Problem: Any authenticated user could access admin routes
Solution: Added profile lookup and role verification in middleware
Impact: Critical security vulnerability fixed

Challenge 2: Missing RLS Policies
Problem: Admin tables had RLS enabled but no policies defined
Solution: Created comprehensive policies for admin-only access
Impact: Database security properly configured

Challenge 3: React State Race Conditions
Problem: Multiple simultaneous updates causing UI flicker
Solution: Implemented optimistic updates with rollback
Impact: Smooth, responsive user experience

Challenge 4: Large List Performance
Problem: Rendering 1000+ users caused lag
Solution: Added virtual scrolling and memoization
Impact: 60% reduction in render time

Challenge 5: Real-time Subscription Memory Leaks
